📖 使用说明

1. 启动服务

双击 start.bat 一键启动 或手动: set UPSTREAM_API_KEY=你的密钥 && python main.py

2. 让其他人访问（内网穿透）

如果需要让其他人从他们的电脑访问，需要做内网穿透：

# 方式一：localtunnel（免费，无需注册） npm install -g localtunnel lt --port 8800 # 会显示公网地址如 https://xxx.loca.lt # 方式二：ngrok（更稳定，需免费注册） # 1. 访问 https://ngrok.com 注册 # 2. ngrok config add-authtoken YOUR_TOKEN # 3. ngrok http 8800

3. 给用户分配子Key

在「用户管理」页面创建用户，系统会自动生成 sk-xxx 格式的子Key，把这个子Key给对方即可。

4. 用户调用方式 — OpenAI 格式

from openai import OpenAI client = OpenAI( base_url="http://你的公网地址/v1", api_key="sk-用户的子Key" ) response = client.chat.completions.create( model="mimo-v2.5", messages=[{"role": "user", "content": "你好"}] )

5. 用户调用方式 — Claude 格式

import anthropic client = anthropic.Anthropic( base_url="http://你的IP:8800", api_key="sk-用户的子Key" ) response = client.messages.create( model="claude-sonnet-4-20250514", max_tokens=1024, messages=[{"role": "user", "content": "你好"}] )

5. curl 示例

# OpenAI 格式 curl http://你的IP:8800/v1/chat/completions \ -H "Authorization: Bearer sk-用户的子Key" \ -H "Content-Type: application/json" \ -d '{"model": "gpt-4", "messages": [{"role": "user", "content": "你好"}]}' # Claude 格式 curl http://你的IP:8800/v1/messages \ -H "x-api-key: sk-用户的子Key" \ -H "Content-Type: application/json" \ -H "anthropic-version: 2023-06-01" \ -d '{"model": "claude-sonnet-4-20250514", "max_tokens": 1024, "messages": [{"role": "user", "content": "你好"}]}'

6. 环境变量

变量说明默认值
UPSTREAM_API_KEY你的真实 API 密钥（OpenAI 或 Claude）（必填）
OPENAI_BASE_URLOpenAI 上游地址https://token-plan-cn.xiaomimimo.com/v1
CLAUDE_BASE_URLClaude 上游地址https://token-plan-cn.xiaomimimo.com/anthropic
CLAUDE_VERSIONClaude API 版本2023-06-01
ADMIN_TOKEN管理面板密码admin123
PROXY_PORT代理服务端口8800

7. 工作原理

代理自动检测用户请求的协议格式：

  • 有 x-api-key header 或访问 /v1/messages → Claude 协议
  • 有 Authorization: Bearer header → OpenAI 协议

用户的子Key会被替换成你的真实密钥后转发到上游 API，所有提示词和回复都会被记录。
